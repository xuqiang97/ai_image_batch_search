# 0. 最关键的前置说明

本需求不是改造现有普通搜索框，也不是新增展开式大输入区域。

本期目标是在 AI 生图 4 个核心列表页中，**新增独立「批量」筛选字段**，复用系统现有“字段下拉 + 多值输入 + 浮层批量输入”组件能力。现有普通搜索保持原样，不字段化、不改变原有查询逻辑。

页面展示上字段名统一精简为「批量」；需求文档中可称为「批量搜索字段」。

本期不做以下能力：

| 不纳入范围 | 说明 |
| --- | --- |
| 全 AI 生图所有页面接入 | 本期仅接入 4 个核心列表页 |
| 改造普通搜索框 | 普通搜索保持现状 |
| 普通搜索与批量搜索互斥 | 两者作为独立筛选条件共同生效 |
| 批量导出 / 批量审核 / 批量推送 | 本需求仅做查询增强 |
| 批量搜索历史记录 | 后续如有需要另提 |
| 记住上次选择字段 | 本期不做前端 localStorage，二期再考虑 |
| 超 500 自动分批查询 | 不自动分批、不自动截断 |
| 查看已完成生图支持 BatchID | 本期暂不支持，保持与现有单个搜索范围一致 |

# 1. 背景和目标

当前 AI 生图列表页已有普通搜索能力，支持用户输入单个关键词快速查询任务。但在运营和美工实际使用中，经常存在按一批 TaskID、ASIN、SKU、BatchID 批量定位任务的场景。

尤其运营侧存在组合查询诉求，例如：普通搜索输入店铺编码，再通过新增批量字段输入一批 ASIN，从而查询指定店铺下的一批 ASIN 任务。现有单个搜索框虽然可以快速搜索店铺 / ASIN / 任务 ID，但无法同时满足“店铺 + 多个 ASIN”这种组合查询场景。

本期目标：

1. 在 4 个核心列表页新增独立「批量」筛选字段，支持字段化多值精确查询。
2. 普通搜索保持现状，降低对老功能的影响。
3. 批量搜索组件复用现有组件形态，减少前端开发和用户学习成本。
4. 支持普通搜索与批量搜索共同生效，补充组合查询能力。
5. 沉淀轻量可复用组件，后续其他列表页可按字段配置继续接入。

# 3. 批量搜索字段配置

## 3.1 字段默认值

**TaskID**

# 5. 批量组件交互规则

## 5.1 组件形态

「批量」字段复用系统现有多值搜索组件形态：

| 组成 | 说明 |
| --- | --- |
| 字段下拉 | 用户选择一个批量搜索字段 |
| 外层输入框 | 支持输入单个值，或多个值用英文逗号分隔 |
| 右侧浮层按钮 | 点击后打开批量输入浮层 |
| 批量输入浮层 | 支持一行一个值，也兼容中英文逗号、Tab 粘贴 |
| 清空 | 清空当前批量搜索值 |
| 应用并查询 | 回写外层输入框，并立即触发当前页面查询 |

## 5.2 外层输入框文案

外层输入框 placeholder：

**多个用英文逗号**

说明：

1. 外层输入框保持紧凑，不展示 500 个上限说明。
2. 500 个上限和详细输入规则在浮层内展示。
3. 外层输入框直接输入内容后，仅更新当前批量搜索条件；输入框失焦不触发查询，需点击页面【查询】或在外层输入框按 Enter 后才触发查询。

## 5.3 批量浮层文案

浮层上方提示：

**兼容中英文逗号、Tab粘贴；空值忽略，重复去重。**

浮层输入框 placeholder：

**一行一个值，最多500个，右下角拖拽可调整高度。**

## 5.4 浮层高度拖拽

批量浮层多行输入框支持用户拖拽右下角调整高度。

规则：

1. 默认高度保持紧凑。
2. 支持纵向拖拽调整高度。
3. 设置最小高度，保证基础可用。
4. 设置最大高度，避免遮挡页面过多内容。
5. 浮层需根据可视区域自动调整位置 / 高度，避免被页面右侧或底部裁切。

## 5.5 外层输入框与浮层联动

外层输入框和浮层为同一个批量字段的两种输入形态：

| 位置 | 展示方式 |
| --- | --- |
| 外层输入框 | `A,B,C` |
| 浮层 textarea | `A` 换行 `B` 换行 `C` |
| 后端查询值 | `["A", "B", "C"]` |

联动规则：

| 场景 | 行为 |
| --- | --- |
| 外层输入 `A,B,C` 后打开浮层 | 浮层展示为三行：A / B / C |
| 浮层输入三行 A / B / C 后点击外部关闭 | 外层回写为 `A,B,C`，不触发查询 |
| 浮层输入后按 Esc 关闭 | 外层回写为英文逗号格式，不触发查询 |
| 浮层点击【应用并查询】 | 外层回写为英文逗号格式，并立即查询 |
| 浮层点击【清空】 | 清空外层和浮层当前输入值，不触发查询 |
| 页面点击【重置】 | 清空普通搜索、批量搜索及其他筛选项，批量字段恢复 TaskID |

## 5.6 字段切换

切换批量搜索字段时，已输入内容保留，沿用现有组件逻辑。

例如用户先选择 ASIN 并输入多个 ASIN，随后切换为 TaskID，输入内容不清空，后续查询按 TaskID 字段解释当前输入值。

# 6. 输入解析规则

## 6.1 分隔符规则

| 输入内容 | 处理方式 |
| --- | --- |
| 英文逗号 `,` | 作为分隔符 |
| 中文逗号 `，` | 兼容为英文逗号 |
| 换行 | 作为分隔符 |
| Tab | 作为分隔符，方便从 Excel 粘贴 |
| 空格 | 不作为分隔符，仅去除前后空格 |
| 空白行 / 连续逗号 | 自动忽略空值 |
| 重复值 | 自动去重 |

补充说明：

1. 去重仅用于减少重复查询值，不改变用户输入值本身。
2. 前端去重前需先 trim 前后空格。
3. 大小写是否视为重复，沿用当前页面对应字段的搜索口径，不额外做大小写转换。
4. 去重后保留用户首次输入顺序，用于外层英文逗号回写和后端传参。

## 6.2 数量上限

单次批量搜索最多支持 **500 个有效值**。

有效值口径：

1. 先按分隔符拆分。
2. 去除前后空格。
3. 过滤空值。
4. 对重复值去重。
5. 去重后的数量作为有效值数量。

## 6.3 超过 500 个的处理

超过 500 个有效值时：

1. 允许录入和保留。
2. 不自动截断。
3. 不自动分批查询。
4. 禁止发起查询。
5. 外层输入框红框提示。
6. 浮层内展示短提示。

浮层内提示文案：

**超出上限：520 / 500**

Toast 或查询拦截提示可同样使用：

**超出上限：520 / 500**

其中 `520` 为当前去重后的有效值数量。

## 6.4 无有效值处理

当浮层内容为空，或输入内容全部为空值 / 空行时，点击【应用并查询】：

1. 不触发查询。
2. 不关闭浮层。
3. 提示：**请输入批量搜索内容**。

# 7. 查询触发规则

## 7.1 页面【查询】

页面【查询】按钮按当前全部筛选条件查询，包括：

1. 当前权限范围。
2. 当前状态 Tab。
3. 普通搜索。
4. 批量搜索。
5. 时间。
6. 创建人。
7. 任务健康度 / 审核状态 / 张数等任务维度筛选。
8. 模型、图片类型、侵权禁售检测等图片维度筛选。
9. 当前页面已有其他筛选条件。

查询后页码回到第 1 页。

若批量搜索值为空，页面【查询】时忽略批量搜索条件，不报错。

## 7.2 浮层【应用并查询】

点击浮层【应用并查询】后：

1. 校验批量搜索有效值。
2. 若有效值为空，提示并不查询。
3. 若有效值超过 500，提示并不查询。
4. 若校验通过，按英文逗号格式回写外层输入框。
5. 关闭浮层。
6. 立即按当前页面全部筛选条件查询。
7. 查询后页码回到第 1 页。

说明：

浮层【应用并查询】等同于“回写批量字段后点击当前页面【查询】”，不是只按批量字段单独查询。

## 7.3 点击外部关闭浮层

用户在浮层中输入内容后，点击浮层外区域关闭：

1. 输入内容同步回写外层输入框。
2. 不触发查询。
3. 后续用户可点击页面【查询】发起查询。

## 7.4 Esc 关闭浮层

用户按 Esc 关闭浮层：

1. 输入内容同步回写外层输入框。
2. 不触发查询。

## 7.5 浮层【清空】

点击浮层【清空】：

1. 只清空当前批量搜索值。
2. 不清空批量字段下拉。
3. 不清空普通搜索。
4. 不清空其他筛选条件。
5. 不触发查询。

## 7.6 页面【重置】

点击页面【重置】：

1. 清空普通搜索。
2. 清空批量搜索。
3. 清空 / 恢复页面其他筛选项，按当前页面原有重置规则执行。
4. 批量字段下拉恢复默认 TaskID。
5. 页码回到第 1 页。
6. 按页面现有重置逻辑刷新。

## 7.7 Enter 行为

| 场景 | Enter 行为 |
| --- | --- |
| 普通搜索单行输入框 | 触发页面查询 |
| 批量外层单行输入框 | 触发页面查询 |
| 批量浮层 textarea | 换行，不触发查询 |

说明：

1. 美工版 AI 生图、运营版 AI 生图当前普通搜索已支持 Enter 查询，本期保持。
2. 美工查看已完成生图、运营查看已完成生图普通搜索本期补齐 Enter 查询。
3. 批量浮层 textarea 内按 Enter 仅换行，需阻止触发外层查询事件。

# 8. 查询匹配规则

## 8.1 任务维度筛选定位

批量搜索属于任务维度筛选条件。命中规则按任务字段判断；任务不命中则整条任务不展示。图片维度筛选仍按当前页面原逻辑执行，仅过滤任务下的图片。

批量搜索仅新增筛选条件，不改变当前页面原有列表展示字段、排序规则、分页规则和任务 / 图片展示结构。

批量搜索结果仍按当前用户权限裁剪；输入值中包含无权限数据时，仅不返回无权限数据，不因该值导致整个查询失败。

## 8.2 普通搜索与批量搜索关系

普通搜索与批量搜索为两个独立筛选条件，**不互斥**。

两者同时填写时，与其他筛选项一起按 **AND** 关系收敛结果。

例如运营版 AI 生图：

| 普通搜索 | 批量字段 | 批量值 | 预期结果 |
| --- | --- | --- | --- |
| Amazon-Z01360-IN | ASIN | B0A, B0B, B0C | 查询当前权限范围内，店铺编码命中 Amazon-Z01360-IN，且 ASIN 命中 B0A / B0B / B0C 任一值的任务 |

## 8.3 批量搜索内部关系

批量字段内部多个值之间为 **OR** 关系。

例如批量字段选择 ASIN，输入：

`B0A,B0B,B0C`

表示：

**ASIN 命中 B0A / B0B / B0C 任一值即可。**

## 8.4 当前 Tab 生效范围

美工版 AI 生图、运营版 AI 生图的批量搜索仅对当前状态 Tab 生效，不跨状态 Tab 查询。

例如用户在“待审核”Tab 下批量搜索 TaskID，仅返回待审核状态下命中的任务。

## 8.5 精确匹配规则

批量搜索按选定字段精确匹配，不支持模糊搜索、前缀搜索、后缀搜索。

大小写处理、字段归一化等底层口径沿用当前页面现有搜索实现，不新增额外转换规则。

本期不做字段格式强校验；用户输入格式不符合实际数据时，按精确匹配自然无结果处理。

## 8.6 店铺字段规则

运营版批量字段「店铺」按店铺编码精确匹配，例如：

`Amazon-Z01360-IN`

不支持按店铺名称、sourceId、店铺 ID 批量搜索。

# 9. 接口与实现建议

## 9.1 前端组件建议

前端建议沉淀为通用轻量批量搜索组件，页面只做字段配置和参数映射。

组件可配置项建议：

| 配置项 | 示例 |
| --- | --- |
| label | 批量 |
| defaultField | TaskID |
| fields | TaskID / BatchID / ASIN / 店铺 |
| maxCount | 500 |
| placeholder | 多个用英文逗号 |
| popoverTip | 兼容中英文逗号、Tab粘贴；空值忽略，重复去重。 |
| textareaPlaceholder | 一行一个值，最多500个，右下角拖拽可调整高度。 |
| normalize | trim、过滤空值、去重、分隔符解析 |
| onQuery | 触发当前页面查询 |
| onReset | 页面重置时清空批量值并恢复默认字段 |

## 9.2 后端接口建议

保留现有普通搜索参数，不改变老逻辑。

新增批量搜索参数建议：

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| batchSearchField | string | 批量搜索字段，例如 taskId / batchId / sku / asin / storeCode |
| batchSearchValues | array<string> | 批量搜索值列表，最多 500 个有效值 |

说明：

1. 普通搜索参数继续沿用现有字段。
2. 批量搜索作为新增查询条件传参。
3. 普通搜索与批量搜索可同时传入，同时传入时按 AND 关系生效。
4. 后端仍需先执行数据权限裁剪，不允许仅依赖前端过滤。
5. 后端需对批量值数量做兜底校验，超过 500 时返回明确错误。
6. 批量搜索值最多 500 个，不建议通过 URL query string 拼接传参；如当前列表接口支持，建议通过 request body 传递 `batchSearchValues`，避免 URL 过长和特殊字符编码问题。
7. 后端批量搜索应基于目标字段做精确匹配查询，避免对批量值逐条循环查询；TaskID、BatchID、系统SKU、ASIN、店铺编码等字段需确认已有索引或可支撑 `IN` 查询，避免批量搜索导致列表接口慢查询。
8. 后端返回超限错误时，前端需能识别并展示统一文案：`超出上限：X / 500`。

# 11. 风险描述

| 风险 | 控制方式 |
| --- | --- |
| 误解为改造普通搜索 | 明确普通搜索保持现状，批量是新增独立字段 |
| 误解为全页面接入 | 明确本期仅接入 4 个核心列表页 |
| 搜索条件叠加导致无结果 | 普通搜索与批量搜索按 AND 生效，验收样例写清楚 |
| 超 500 自动截断导致漏查 | 明确禁止自动截断，超限直接拦截 |
| 页面空间不足 | 主流程页插入搜索与查询之间；已完成页缩短普通搜索框，不新增行 |
| 浮层被裁切 | 前端需根据可视区域处理位置和最大高度 |
| 仅前端过滤导致越权 | 后端必须基于权限范围进行查询裁剪 |
| 批量搜索导致慢查询 | 后端需使用批量精确匹配能力，并确认核心查询字段索引或查询性能 |
| 字段切换不清空导致误查 | 沿用现有组件逻辑；后续如反馈明显再二期优化 |
