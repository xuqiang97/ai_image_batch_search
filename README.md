## 1. 背景与目标

当前 AI 生图列表页已有普通搜索能力，但无法高效支持一批 TaskID、ASIN、SKU、BatchID 的批量定位。

本期在 AI 生图 4 个核心列表页新增独立「批量」筛选字段，现有普通搜索保持原样。

典型场景：

1. **风控复审**：随机抽取各部门的一批任务 ID 给到风控，风控通过批量搜索 TaskID 快速定位任务并进行图片复审。
2. **运营定位**：运营普通搜索输入店铺编码，再通过「批量」字段输入一批 ASIN，查询指定店铺下的一批 ASIN 任务。

目标：

1. 支持字段化多值精确查询。
2. 支持普通搜索与批量搜索共同生效。
3. 复用轻量批量输入组件，后续其他列表页可按字段配置接入。

| 页面 | 批量字段顺序 |

| 美工版 AI 生图 | TaskID / BatchID / 系统SKU |
| 运营版 AI 生图 | TaskID / BatchID / ASIN / 店铺 |
| 美工查看已完成生图 | TaskID / 系统SKU |
| 运营查看已完成生图 | TaskID / ASIN / 店铺 |

规则：

1. 所有页面默认字段均为 **TaskID**，页面【重置】后恢复 TaskID。
2. BatchID 仅在美工版 AI 生图、运营版 AI 生图出现。
3. 查看已完成生图本期不支持 BatchID。
4. 运营版「店铺」按店铺编码精确匹配，例如 `Amazon-Z01360-IN`，不支持店铺名称、sourceId、店铺 ID。

## 4. 批量组件交互

### 4.1 组件文案

| 位置 | 文案 / 规则 |
| --- | --- |
| 字段名 | 批量 |
| 外层输入框 placeholder | 多个用英文逗号 |
| 浮层提示 | 兼容中英文逗号、Tab粘贴；空值忽略，重复去重。 |
| 浮层 textarea placeholder | 一行一个值，最多500个，右下角拖拽可调整高度。 |

外层输入框和浮层 textarea 使用同一套输入解析规则；外层 placeholder 仅做简化提示。

### 4.2 联动规则

| 场景 | 行为 |
| --- | --- |
| 点击右侧图标 | 打开批量输入浮层；浮层已打开时再次点击则关闭 |
| 外层输入 `A,B,C` 后打开浮层 | textarea 按三行展示 A / B / C |
| 浮层输入后点击外部关闭 | 回写外层，不触发查询 |
| 浮层输入后直接点击页面【查询】/ 状态 Tab / 左侧菜单 | 先回写外层，再执行对应动作 |
| 按 Esc | 关闭浮层，回写外层，不触发查询；光标在 textarea 内也生效 |
| 点击【清空】 | 清空当前批量搜索值，不触发查询 |
| 点击【应用并查询】 | 回写外层，并立即按当前页面全部筛选条件查询 |
| 切换批量字段 | 已输入内容保留，不自动查询；后续按切换后的字段解释当前输入值 |
| textarea 按 Enter | 换行，不触发查询 |
| 普通搜索 / 批量外层输入框按 Enter | 触发当前页面查询 |

美工查看已完成生图、运营查看已完成生图普通搜索本期补齐 Enter 查询。

## 5. 输入解析与数量限制

### 5.1 解析规则

| 输入内容 | 处理方式 |
| --- | --- |
| 英文逗号 `,` / 中文逗号 `，` / 换行 / Tab | 作为分隔符 |
| 空格 | 不作为分隔符，仅去除前后空格 |
| 空白行 / 连续逗号 | 自动忽略空值 |
| 重复值 | 自动去重 |

有效值口径：先按分隔符拆分 → trim 前后空格 → 过滤空值 → 去重 → 按去重后的数量计数。

前端去重仅按原始字符串完全一致去重，不主动转小写、不主动转大写；后端 / 数据库匹配时是否大小写敏感，沿用当前页面现有查询口径。

示例：

| 输入 | 预期结果 |
| --- | --- |
| `A,B,C` / `A，B，C` / 三行 A、B、C | 解析为 A / B / C |
| 从 Excel 复制多列或多行 Tab 内容 | 正常解析为多个值 |
| `A,,B` | 空值忽略，解析为 A / B |
| ` A ` | 前后空格去除，解析为 A |
| `A B` | 不按空格拆分，作为一个完整值处理 |
| `A,A,B` | 重复值去重，解析为 A / B |
| `A,a,B` | 不强制转小写去重，解析为 A / a / B |

### 5.2 数量限制

单次批量搜索最多支持 **500 个有效值**。

| 输入 | 去重后有效值数量 | 是否允许查询 |
| --- | ---: | --- |
| `A` 重复 600 次 | 1 | 允许 |
| 501 个不同值 | 501 | 禁止 |

超过 500 个有效值时：允许录入和保留，但禁止查询，不自动截断、不自动分批；提示文案统一为 **超出上限：X / 500**。

用户在浮层 textarea 中编辑内容时，外层输入框红框状态需实时同步：超过 500 时显示红框，恢复到 500 以内时红框消失。

### 5.3 空内容

浮层内容为空，或全部为空值 / 空行时，点击【应用并查询】：不查询、不关闭浮层，提示 **请输入批量搜索内容**。

如需清空批量搜索条件，用户应点击【清空】或页面【重置】。

## 6. 查询规则

| 规则 | 说明 |
| --- | --- |
| 查询触发 | 页面【查询】按当前全部筛选条件查询；浮层【应用并查询】等同于回写批量字段后点击当前页面【查询】 |
| 页码 | 查询后回到第 1 页 |
| 批量为空 | 忽略批量搜索条件，不报错 |
| 普通搜索 与 批量搜索 | AND |
| 批量字段内部多个值 | OR |
| 批量搜索 与 其他筛选项 | AND |
| 结果排序 | 沿用当前页面原有排序规则，不按批量值输入顺序排序 |
| 权限 | 结果按当前用户权限裁剪；无权限数据不返回，不导致整个查询失败 |
| 无结果 | 按当前页面原有空状态展示，不报错 |

示例：

| 普通搜索 | 批量字段 | 批量值 | 预期结果 |
| --- | --- | --- | --- |
| Amazon-Z01360-IN | ASIN | B0A, B0B, B0C | 查询当前权限范围内，店铺编码命中 Amazon-Z01360-IN，且 ASIN 命中 B0A / B0B / B0C 任一值的任务 |

生效范围：

| 页面 | 生效范围 |
| --- | --- |
| 美工版 AI 生图 / 运营版 AI 生图 | 仅对当前状态 Tab 生效 |
| 美工查看已完成生图 / 运营查看已完成生图 | 仅对当前图片状态子 Tab 生效，包括全部图片 / 通过图片 / 不通过图片 |

匹配规则：

1. 批量搜索按选定字段精确匹配。
2. 不支持模糊搜索、前缀搜索、后缀搜索。
3. 本期不做字段格式强校验，输入不符合实际数据时自然无结果。

## 7. 接口与实现建议

保留现有普通搜索参数与查询逻辑，批量搜索仅作为新增筛选条件接入；具体接口结构由后端研发按现有列表接口规范确定。

实现规则：

1. 普通搜索与批量搜索同时存在时按 AND 生效；批量字段内部多个值按 OR 生效。
2. 批量搜索值为空时，后端统一视为不启用批量搜索条件。
3. 批量字段必须通过后端白名单 / 枚举映射到实际查询字段，不允许直接拼接前端展示字段或前端传入字段名。
4. 后端需按同一有效值口径兜底处理：trim、过滤空值、去重、计数；最多 500 个有效值。
5. 后端必须基于当前登录用户权限进行数据裁剪，不能仅依赖前端过滤。
6. 批量搜索按目标字段精确匹配，避免逐条循环查询；TaskID、BatchID、系统SKU、ASIN、店铺编码等核心字段需确认索引或查询性能。
7. 超过 500 个有效值时，前端统一展示：`超出上限：X / 500`；后端返回内容需能支持前端识别当前数量和上限。
8. 批量值较多，不建议通过 URL query string 拼接传参；具体传参方式由研发结合现有列表接口规范确定。

| AI 生图主流程页查询按钮 | 调整为蓝色主按钮 |

### 8.2 交互与查询

| 场景 | 预期结果 |
| --- | --- |
| 外层输入 `A,B,C` 后打开浮层 | textarea 按三行展示 A / B / C |
| 浮层输入后点击外部 / 按 Esc / 再次点击右侧图标 | 回写外层，不触发查询 |
| 光标在 textarea 内时按 Esc | 关闭浮层，回写外层，不触发查询 |
| 浮层输入后直接点击页面【查询】/ 状态 Tab / 左侧菜单 | 先回写外层，再执行对应动作 |
| 批量字段切换 | 已输入内容保留，不自动查询 |
| 浮层点击【清空】 | 清空批量值，不触发查询 |
| 浮层点击【应用并查询】 | 回写外层并立即查询 |
| 普通搜索 / 批量外层输入框按 Enter | 触发当前页面查询 |
| textarea 内按 Enter | 换行，不触发查询 |
| 仅普通搜索有值 | 按原普通搜索逻辑查询 |
| 普通搜索有值，批量搜索为空 | 仅按普通搜索及其他筛选条件查询，不报错 |
| 普通搜索和批量搜索都有值 | 两者 AND 共同收敛结果 |
| 批量搜索内部多个值 | OR 命中任一值即可 |
| 主流程页 / 已完成页切换 Tab 后查询 | 仅查询当前 Tab / 子 Tab 范围内的数据 |
| 批量搜索值均未命中数据 | 按当前页面原有空状态展示，不报错 |
| 点击页面【查询】或浮层【应用并查询】 | 当前全部筛选条件生效，页码回第 1 页 |

### 8.3 异常

| 异常场景 | 预期结果 |
| --- | --- |
| 有效值 501 个 | 禁止查询，提示 `超出上限：501 / 500` |
| 外层或浮层输入 501 个有效值后查询 | 先保留 / 回写内容，外层红框提示，禁止查询，并提示 `超出上限：501 / 500` |
| 浮层 textarea 输入超过 500 个有效值 | 浮层提示超限，外层输入框同步红框 |
| 浮层 textarea 从超过 500 个删回 500 个以内 | 浮层提示恢复正常，外层输入框红框同步消失 |
| 超限后自动截断 / 自动分批查询 | 不允许 |
| 空内容点击【应用并查询】 | 不查询、不关闭浮层，提示 `请输入批量搜索内容` |
| 店铺字段输入店铺名称 | 不按店铺名称匹配，仅按店铺编码匹配 |
| `batchSearchField` 非白名单字段 | 后端返回参数错误，前端按页面现有错误提示方式展示，不发生服务异常 |

## 9. 权限与风险

本需求不新增菜单、不新增角色、不新增按钮级权限。用户能访问对应页面，即可使用该页面内的「批量」筛选字段。

数据权限完全沿用当前页面原有权限规则，后端查询必须继续基于当前登录用户权限进行数据裁剪。

主要风险及控制方式：

| 风险 | 控制方式 |
| --- | --- |
| 误解为改造普通搜索 | 明确普通搜索保持现状，批量是新增独立字段 |
| 搜索条件叠加导致无结果 | 普通搜索与批量搜索按 AND 生效，验收样例写清楚 |
| 超 500 自动截断导致漏查 | 明确禁止自动截断，超限直接拦截 |
| 仅前端过滤导致越权 | 后端必须基于权限范围进行查询裁剪 |
| 批量搜索导致慢查询 | 后端需使用批量精确匹配能力，并确认核心字段索引或查询性能 |
